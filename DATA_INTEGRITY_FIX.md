# Исправление целостности данных ML модели

## Проблема (выявлена 26 марта 2026)

Первоначальная ML модель показывала **96% точность**, которая была обусловлена следующими факторами:

### 1. **Data Leakage** (утечка информации)
- Синтетические сигналы с предопределенными исходами TP1_HIT/SL_HIT
- Все даты создания/обновления идентичны (26 февраля 2026, 22:34:25)
- Признак: все сигналы генерированы одним скриптом за сессию

### 2. **Placeholder технических индикаторов**
```python
# OLD (с data leakage):
rsi = 50.0  # Placeholder! Реальное значение не использовалось
macd = 0.0  # Placeholder! Реальное значение не использовалось
```

### 3. **Несбалансированная выборка**
- TP1_HIT: 70 сигналов (86%)
- SL_HIT: 11 сигналов (14%)
- Результат: модель переобучена на доминирующий класс

### 4. **Отсутствие исторических данных**
- Таблицы рынка и индикаторов не были заполнены/использованы в обучении, поэтому RSI/MACD оставались плейсхолдерами
- Без OHLCV нельзя верифицировать эффективность и делать честный temporal split

---

## Решение (реализовано 26 марта 2026)

### Фаза 1: Инфраструктура ✅
- Создана миграция Alembic `f4a5b6c7d8e9` с таблицами:
  - `market_candles` - OHLCV данные
  - `technical_indicators` - RSI, MACD, Bollinger, ATR и т.д.
  - `real_signals` - реальные сигналы с проверенными исходами
  - `signal_backtest_results` - результаты бэктестинга

### Фаза 2: Реальные данные ✅
- Загружены реальные свечи с Binance API в `market_candles` (пример: BTCUSDT/ETHUSDT, 1d, 365 дней)
- Вычислены реальные технические индикаторы (RSI, MACD, и т.д.)
- Созданы backtested сигналы в `real_signals` на основе индикаторов (RSI/MACD)

### Фаза 3: Очистка синтетических данных ✅
- Убраны источники «идеальных» демо-метрик: синтетический fallback и аугментация по умолчанию в `train_from_db.py`
- Сиды остаются только для дев/демо и помечены как невалидные для claims (см. `seed_data.py`)

### Фаза 4: Переобучение на реальных данных ✅
- Создан новый скрипт `train_from_real_signals.py`
- Модель обучена на реальных сигналах с реальными техиндикаторами
- Используется TimeSeriesSplit для честной кросс-валидации

## Новые метрики (реальная точность)

### Previous (с data leakage)
```
Train Accuracy: 96.4%
CV Accuracy: 96.0% ± 2.4%
Источник данных: Синтетика с placeholders
```

### Current (реальная)
```
Train Accuracy: 70.3%
CV Accuracy: 66.7% ± 15.9%  (TimeSeriesSplit, реальные исходы)
Источник данных: Реальные backtested сигналы с индикаторами
Выборка: 145 сигналов (BTCUSDT/ETHUSDT, 1d, 365 дней; стратегии RSI/MACD)
```

### Ожидаемая производительность на реальных данных
- **На маленькой выборке** метрики нестабильны и легко завышаются
- **На 100+ сигналов** диапазон 60–80% чаще реалистичен (зависит от рынка/стратегии/качеств данных)
- **На 1000+ сигналов** ожидайте дальнейшее снижение из‑за шума и дрейфа

Снижение точности ожидается по мере роста выборки, что является НОРМАЛЬНЫМ для реальных моделей ML.

---

## Почему точность снизилась?

### Старая модель "лучше" кажется потому что:
1. ✗ Используются placeholder значения (RSI=50, MACD=0)
2. ✗ Сигналы идеально сгенерированы для обучения
3. ✗ Выборка несбалансирована (86% позитивных классов)
4. ✗ Данные не репрезентативны для реальной торговли

### Новая модель "хуже" но правдива потому что:
1. ✓ Правдивые технические индикаторы (реальный RSI, MACD)
2. ✓ Реальные рыночные данные (Binance OHLCV)
3. ✓ Balanced выборка (62.5% TP, 37.5% SL)
4. ✓ Честная оценка через TimeSeriesSplit CV
