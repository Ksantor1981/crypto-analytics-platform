# План работ: целостность данных и честный ML

## Проблема (корневая)

1. **Таблица `signals`** может быть заполнена **сидами** (`seed_data.py`) — одинаковые временные метки, нет связи с рынком.
2. **`train_from_db.py`** исторически использовал **RSI=50 / MACD=0**, **фиксированный `price_dev`**, **аугментацию** до 500 строк и **синтетический fallback** — метрики CV **не отражают** реальную торговлю.
3. **Таблицы рынка** (`market_candles`, `technical_indicators`, `real_signals`) есть в миграции `f4a5b6c7d8e9_*`, но **пайплайн до обучения** часто не запускают.
4. В маркетинге и SPEC фигурировала **87.2%** как факт, а не **целевой KPI** — это вводит в заблуждение.

## Переменные окружения (обучение)

| Переменная | Значение по умолчанию | Смысл |
|------------|----------------------|--------|
| `ML_AUGMENT_TARGET` | `0` | `>0` — ауgment до N образцов (**только для экспериментов**). |
| `ML_SYNTHETIC_FALLBACK` | не задано | `1` — разрешить старый синтетический fallback при малых N. |
| `ML_INDICATOR_TIMEFRAME` | `1d` | Таймфрейм для JOIN к `technical_indicators`. |
| `ML_STRICT_DATA_QUALITY` | не задано | `1` — выход с ошибкой при низком покрытии индикаторов. |
