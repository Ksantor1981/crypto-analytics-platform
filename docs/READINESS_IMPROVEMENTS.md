## 3. Пользовательские сценарии (end-to-end)

### Сценарий A: Новый пользователь (бесплатный)

1. Регистрация → подтверждение email (если настроено)
2. Вход
3. Главная → Дашборд (пусто или подсказка)
4. Каналы — список (без добавления)
5. Рейтинги — топ/антирейтинг
6. Сигналы — 0 (нужен вход) или подсказка

**Пробелы:** Регистрация, лимиты (mock).

### Сценарий B: Подписка Pro

1. Вход
2. Тарифы → «Выбрать Pro»
3. Stripe Checkout → оплата (test card)
4. Webhook → обновление подписки
5. Профиль — отображается Pro план

**Проверка Stripe Checkout (test mode):**
1. Backend: `STRIPE_SECRET_KEY=sk_test_...` в `.env`
2. Вход → /subscription → «Выбрать Pro» → редирект на Stripe
3. Тестовая карта: 4242 4242 4242 4242
4. Webhook требует ngrok/публичный URL для обновления подписки

### Сценарий C: Анализ сигнала

1. Вход
2. Сигналы → список
3. Кнопка «ML Анализ» на сигнале
4. Результат предсказания

**Пробелы:** ML-сервис на 8001, fallback если недоступен.

### Сценарий D: Добавить свой канал

1. Вход
2. Каналы → «Добавить канал»
3. URL (t.me/...) → сохранение

**Статус:** Реализовано.
