# Аудит задач ТЗ: готовность и работоспособность

**Источник:** корневой [`SPEC.md`](../SPEC.md).  
**Метод:** сверка структуры репозитория, кода, CI, доков; полный e2e на всех стендах здесь не выполнялся.

**Условные обозначения**

| Символ | Значение |
|--------|----------|
| **Г** | Есть в репозитории, типичный сценарий реализован |
| **Ч** | Частично: есть база, критерий ТЗ не закрыт полностью |
| **Н** | Нет или не подтверждено |
| **П** | Зависит от эксплуатации (VPS, прогоны, бизнес) |
| **Т** | Есть автопроверки (pytest / CI / tsc / jest) — не гарантия всех сценариев |

**Рекомендуемая регрессия после изменений**

```powershell
cd backend && python -m pytest tests/ --ignore=tests/test_z_production_integration.py -q
cd frontend && npm run type-check && npm test && npm run build
```

## Итог

- **Ядро продукта** (API, БД, сбор, фронт основных страниц, Stripe, ML-сервис как компонент) — **в основном Г**, с автотестами на backend и базовыми на frontend.

- **Известный пробел UX/API:** экспорт Premium — **бэкенд есть**, **клиент `api.ts` не обнаружен вызовом export** при быстром аудите — стоит добавить кнопки/методы.
