# Замыкание жизненного цикла сигнала («truth layer»)

**Цель:** превратить сбор сигналов в **измеримую петлю**: событие → цена/исход → метрики → обучение/отчёты.

## Текущее состояние (кратко)

- Сигналы сохраняются с полями входа, TP/SL, статусами (`Signal`), пересчёт метрик каналов (`metrics_calculator`, `signal_checker`).
- Часть строк может долго оставаться в `PENDING` при неполном парсинге уровней или отсутствии котировки по активу — см. пайплайн и `docs/PIPELINE_SIGNALS.md`.

## Что добавить для «продуктовой правды»

| Слой | Действие | Зачем |
|------|-----------|--------|
| **Данные** | Явные поля/витрина: время сигнала, entry, исход (TP1/SL/EXPIRED), при необходимости снимок цены на горизонтах 1h/24h/7d | Win rate, ROI, precision по каналу |
| **Пайплайн** | Не допускать «вечных» PENDING без политики: нет TP/SL → EXPIRED/CANCELLED или доработка парсера | Честная статистика |
| **ML** | Только размеченные исходы в обучающей выборке; версионирование датасета | См. `ML_DATA_INTEGRITY_ROADMAP.md` |
| **Отчётность** | Экспорт и дашборды: согласованные определения метрик (один источник правды) | Доверие инвестора/заказчика |
