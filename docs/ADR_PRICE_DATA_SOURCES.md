# ADR: источники рыночных цен (CoinGecko vs биржи из ТЗ)

**Статус:** принято (вариант A — честное соответствие документации)  
**Дата:** 2026-04-03  

## Контекст

В `SPEC.md` в разделе интеграций упомянуты **Binance** и **Bybit**. В коде первичная валидация и текущие котировки для большинства сценариев берутся из **CoinGecko** (`backend/app/services/price_validator.py` и связанная логика).

## Решение

- **Фактическая архитектура:** внешний источник котировок для MVP — **CoinGecko** (с кэшем Redis при наличии).
- **Формулировка ТЗ:** трактовать как «внешние рыночные данные»; при строгой приёмке — **обновить SPEC** явной строкой: *«primary: CoinGecko; опционально REST Binance/Bybit для бэктеста/валидации»*.

## Альтернатива (вариант B)

Добавить тонкие REST-клиенты Binance/Bybit и цепочку fallback (например CoinGecko → Binance → Bybit) для критичных путей — отдельная задача с тестами и лимитами API.

## Последствия

- Приёмка «буквально по старому тексту ТЗ» без правки SPEC формально показывает разрыв; продуктовое решение — **либо правка ТЗ**, **либо доработка интеграций**.
