# Приложение к ТЗ: приёмка канонического data plane

**Статус:** рабочий чек-лист для сверки с ТЗ (`SPEC.md`, дополнение «канонический data plane»).  
**Жизненный цикл:** когда **все обязательные пункты** ниже закрыты, артефакты обновлены (`SPEC_COMPLIANCE_*.md`, при необходимости `SPEC.md`) — **этот файл удаляют** из репозитория (или архивируют в `docs/archive/` по политике команды).

**Первичные ссылки:**  
`DATA_PLANE_MIGRATION.md` · `MARKET_OUTCOME_POLICY.md` · `DOMAIN_GLOSSARY_CANONICAL.md` · `REVIEW_GUIDELINES.md` · `PROD_REMEDIATION_PLAN.md`

---

## Как пользоваться при проверке соответствия ТЗ

1. Пройти блоки **A→J** сверху вниз; для каждого пункта зафиксировать доказательство (команда, путь к тесту, скрин, отчёт, PR).
2. Раз в спринт (или перед релизом) заполнять таблицу **«История проверок»** внизу.
3. Расхождения между этим файлом и кодом: либо исправить код/доки, либо завести явное исключение в `SPEC_COMPLIANCE_*` с датой и причиной.

**Легенда:** `[ ]` не готово · `[x]` готово · `N/A` не применимо к текущему контуру
