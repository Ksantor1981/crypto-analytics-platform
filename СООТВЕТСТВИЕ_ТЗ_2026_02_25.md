## Docker

**Запуск:** из папки проекта `cd crypto-analytics-platform`, затем `docker compose up -d`.

**Если backend unhealthy (password auth failed):** том postgres инициализирован со старым паролем. Исправление:
```powershell
cd c:\project\crypto-analytics-platform
docker compose down
docker rm -f crypto-analytics-backend  # при необходимости перезапусти Docker Desktop
docker volume rm crypto-analytics-platform_postgres_data
docker compose up -d postgres redis backend ml-service frontend
```

**Проверка `.env`:** `POSTGRES_PASSWORD` и пароль в `DATABASE_URL` должны совпадать.

**Минимальный стек:** `docker compose up postgres redis -d`  
**Celery beat** — в `docker-compose.production.yml`. Для автосбора хватает APScheduler в backend.
